## 3. Generation Method

### 3.1 Principle

The generated schema artifact is produced by concatenating every **forward** migration file in the canonical directory in ascending lexicographic order of full file name. This matches the ordering rule declared in `MIGRATION_NAMING_AND_ORDERING_STANDARD.md` §5 and recorded in `D-P2-01` §4.

### 3.2 Tool

- **Tool:** PowerShell (no external CLI dependencies).
- **Reason:** The environment has `supabase` and `docker` available, but a live PostgreSQL instance is not required for a source-level canonical-chain artifact. Concatenation is the only method that is fully deterministic and reproducible without a running database.

### 3.3 Exclusions

- **Rollback files:** Any `.sql` file under `supabase/migrations/rollback/` is excluded because reverse migrations are not part of the forward canonical chain.
- **Orphan files:** Any `.sql` file outside `supabase/migrations` is excluded, per `D-P2-02_Orphan_SQL_Triage_Record.md`.

### 3.4 Traceability Wrappers

Each migration is wrapped with comments that record:

```sql
-- === MIGRATION START: <file_name> ===
-- SOURCE_PATH: <absolute_path>
-- SIZE_BYTES: <size>
-- SHA256: <sha256_hash>

<migration_content>

-- === MIGRATION END: <file_name> ===
```

These wrappers make the artifact self-describing and allow automated verification that each embedded migration body matches its source file.

## 5. Generation Procedure

The artifact was generated by the following PowerShell procedure.

```powershell
$repo    = "C:\Users\SUACAUBA\Downloads\Project\vietsale-pro-v7"
$out     = Join-Path (Join-Path $repo "supabase") "schema.sql"
$migDir  = Join-Path (Join-Path $repo "supabase") "migrations"

# Only forward migrations in the canonical directory root
$files = Get-ChildItem -Path $migDir -Filter "*.sql" -File | Sort-Object Name

$stream = [System.IO.StreamWriter]::new($out, $false, [System.Text.Encoding]::UTF8)

$stream.WriteLine("-- Generated Schema Artifact")
$stream.WriteLine("-- Program: VietSalePro v7 — System Recovery Program")
$stream.WriteLine("-- Deliverable: D-P2-03 — Generated Schema Artifact")
$stream.WriteLine("-- Generation method: Concatenation of canonical forward migrations")
$stream.WriteLine("-- Canonical source directory: supabase/migrations")
$stream.WriteLine("-- This file is produced solely from the canonical migration chain defined in D-P2-01.")
$stream.WriteLine("-- No orphan SQL files were used as source.")
$stream.WriteLine("-- Migration count: $($files.Count)")
$stream.WriteLine()

foreach ($f in $files) {
    $sha = (Get-FileHash -Path $f.FullName -Algorithm SHA256).Hash
    $stream.WriteLine("-- === MIGRATION START: $($f.Name) ===")
    $stream.WriteLine("-- SOURCE_PATH: $($f.FullName)")
    $stream.WriteLine("-- SIZE_BYTES: $($f.Length)")
    $stream.WriteLine("-- SHA256: $sha")
    $stream.WriteLine()

    $content = Get-Content -Path $f.FullName -Raw -Encoding UTF8
    # Normalize line endings to LF for deterministic artifact
    $content = $content -replace "\r\n", "`n"
    $stream.Write($content)

    if (-not $content.EndsWith("`n")) { $stream.WriteLine() }
    $stream.WriteLine("-- === MIGRATION END: $($f.Name) ===")
    $stream.WriteLine()
}

$stream.Close()
```

**Command executed:** The above PowerShell block was run in the project root.
