# PHASE_01_CODEBASE_MEMORY_INITIALIZATION

## Program

VietSalePro -- Codebase Memory Initialization

### Objective

Build a complete semantic knowledge base for VietSalePro using
Codebase-memory-MCP. This task is READ ONLY.

## Background

The purpose is to create reusable semantic knowledge for future
implementation, debugging, governance and planning. The goal is to
understand the software as a system, not merely index files.

## Required MCP

-   **Mandatory:** Codebase-memory-MCP
-   **Read-only if available:** Supabase MCP, Vercel MCP

## Required Skills

-   codebase-design
-   writing-plans
-   doc-coauthoring

## Execution Mode

READ ONLY. No code changes, commits, deployments, migrations or database
modifications.

## Scope

Index and understand: - Project structure and dependency graph -
Application architecture - Business modules and responsibilities -
Business logic, validation, workflows - UI, routes, layouts, hooks,
contexts - Services, utilities, APIs, DTOs - Database schema, tables,
views, triggers, indexes, RLS - Migration history - RPC functions - Edge
Functions - Environment and deployment configuration - Tests and mocks -
Governance and documentation

## Semantic Memory Requirements

Build semantic memory instead of only file-level memory.

Preserve: - Business intent - Architectural intent - Module
responsibilities - Cross-module relationships - Business workflows -
Data flow - Dependency graph - Naming conventions - Design patterns -
Domain terminology - Historical evolution where discoverable

## Knowledge Graph

Construct relationships: Business Module -\> Service -\> Database -\>
RPC -\> Edge Function -\> UI -\> Route -\> Permission -\> Workflow -\>
Reports

## Forbidden

-   No bug investigation
-   No recommendations
-   No implementation
-   No quality evaluation
-   No repository modifications

## Evidence

Report: - Indexed domains - Modules - Services - Tables - RPCs - Edge
Functions - Migrations - Architecture summary - Confidence level -
Anything not indexed

## Exit Criteria

Codebase Memory can answer future questions without rescanning the
repository whenever possible.
