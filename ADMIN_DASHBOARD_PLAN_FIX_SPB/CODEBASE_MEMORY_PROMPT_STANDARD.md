# VietSalePro — ChatGPT Prompt Standard for Codebase Memory

> **Date**: 2026-07-24
> **Authority**: Chief Technical Advisor / Production Owner

## 1. Standard Wording

Future prompts that mention `.codebase-memory` shall use the following description:

> `.codebase-memory` is a generated AI Workspace at the repository root, produced and consumed by the `codebase-memory` MCP. It is not Application Source and is not committed to Git by default. Its binary artifact (`graph.db.zst`) and metadata (`artifact.json`) are regenerated by `codebase-memory.index_repository`. Markdown artifacts (`SEMANTIC_MEMORY.md`, `VALIDATION_REPORT.md`, `CODEBASE_MEMORY_BASELINE.md`) are documentation outputs and, if retained, belong in an approved curated directory. Do not relocate `.codebase-memory`; the MCP does not support alternate storage locations.

## 2. Required Phrases

Include at least one of the following in every Codebase Memory governance prompt:

- `.codebase-memory` is an AI Workspace generated by the `codebase-memory` MCP.
- `.codebase-memory` is bound to the repository root and is not relocatable.
- Generated artifacts inside `.codebase-memory` shall not be committed unless explicitly authorized.
- The repository source is the ultimate source of truth; `.codebase-memory` is a semantic cache.

## 3. Prohibited Instructions

Do not instruct an agent to:

- Relocate `.codebase-memory` outside the repository.
- Commit `graph.db.zst` or `artifact.json` to Git.
- Hand-edit `graph.db.zst` or `artifact.json`.
- Treat `.codebase-memory` files as unauthorized without verifying whether they were generated by the MCP.

## 4. Example Prompt Snippet

```
Project: VietSalePro
Codebase Memory: `.codebase-memory/` at repository root, generated by codebase-memory MCP, not committed by default.
Governance: see CODEBASE_MEMORY_GOVERNANCE_BASELINE.md and CODEBASE_MEMORY_GIT_POLICY.md.
Authority: Production Owner must approve any relocation or committed baseline snapshot.
```
