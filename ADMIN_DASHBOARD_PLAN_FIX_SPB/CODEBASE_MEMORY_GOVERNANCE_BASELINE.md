## 1. Scope & Authority

This baseline governs how `.codebase-memory` is created, classified, inspected, finalized, and used by Engineering Execution Agents and future ChatGPT sessions. It is the official reference for repository governance, Git operations, and AI Workspace behavior.

## 2. Asset Classification

| Asset | Path | Classification | Justification |

| AI Workspace | `.codebase-memory/` | AI Workspace | Generated by the `codebase-memory` MCP for AI-agent semantic context. Not human-authored source code. |

## 3. Repository Assets

Repository assets are the human-authored and curated files under Git that represent source code, migrations, tests, configuration, and approved documentation. They are the **ultimate source of truth**.

## 4. AI Workspace

`.codebase-memory/` is the AI Workspace. It is bound to the repository root, as confirmed by `codebase-memory` MCP `list_projects` and the Relocation Audit. The MCP does not support an alternate storage location. The workspace is regenerated by the MCP; it is not a hand-maintained directory.

## 5. Generated Runtime Artifacts

The following are produced by the MCP or update scripts and are not authoritative source:

- `graph.db.zst`
- `artifact.json`
- `update-codebase-memory.txt`

These must not be edited by hand, must not be committed by default, and must be regenerated by the MCP.

## 6. Documentation Artifacts

`SEMANTIC_MEMORY.md`, `VALIDATION_REPORT.md`, and `CODEBASE_MEMORY_BASELINE.md` are documentation artifacts. They may be preserved as governance records only if relocated to a non-generated path (for example, `ADMIN_DASHBOARD_PLAN_FIX_SPB/` or `docs/`). In `.codebase-memory/` they are considered generated outputs.

## 7. Version-Controlled Assets

Only files that are human-authored, human-curated, or explicitly approved by the Production Owner belong under Git. `.codebase-memory` artifacts do not meet this test by default.

## 8. Ignored Runtime Assets

`.codebase-memory/` and `memory-zone/.codebase-memory/` are runtime-generated AI workspaces. They shall be ignored by Git unless the Production Owner explicitly authorizes a baseline snapshot in a different directory.

## 9. Repository Inspection Rules

- Routine inspection shall ignore `.codebase-memory/` files.
- While `.codebase-memory` files remain tracked or modified in the working tree, they are treated as governance exceptions and reported as warnings, not source-code errors.
- No inspection may treat a regenerated `.codebase-memory` file as an unauthorized source modification unless it was hand-edited without authority.

## 10. Repository Finalization Rules

- `.codebase-memory` generated artifacts shall not be included in finalization commits.
- If `.codebase-memory` files are tracked or modified at finalization, finalization is blocked until they are ignored, reverted, or explicitly authorized by the Production Owner.
- Relocated baseline markdown snapshots may be included in finalization only if they reside in an approved documentation path and are explicitly authorized.
