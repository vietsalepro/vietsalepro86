# VietSalePro — Codebase Memory Engineering Execution Standard

> **Date**: 2026-07-24
> **Audience**: Engineering Execution Agents

## 1. What `.codebase-memory` Is

`.codebase-memory` is the AI Workspace generated by the `codebase-memory` MCP. It is bound to the repository root and is not relocatable. It provides semantic context for AI agents but is not the source of truth.

## 2. What Agents May Modify

- Agents may read `.codebase-memory/SEMANTIC_MEMORY.md`, `.codebase-memory/VALIDATION_REPORT.md`, and `.codebase-memory/CODEBASE_MEMORY_BASELINE.md` for context.
- Agents may edit `.codebase-memory/SEMANTIC_MEMORY.md` and `.codebase-memory/CODEBASE_MEMORY_BASELINE.md` only when an explicit `CODEBASE MEMORY INCREMENTAL UPDATE` instruction is issued, and only to reflect approved source changes.
- Agents may never hand-edit `graph.db.zst`, `artifact.json`, or `update-codebase-memory.txt`.

## 3. What Agents May Commit

- Agents may not commit any file inside `.codebase-memory/` unless the Production Owner explicitly authorizes the commit and the file has been relocated to an approved documentation directory.
- Agents may not add, remove, or alter `.gitignore` entries for `.codebase-memory` unless explicitly authorized.

## 4. What Shall Never Be Considered Unauthorized

- Changes to `.codebase-memory/graph.db.zst`, `.codebase-memory/artifact.json`, and `.codebase-memory/update-codebase-memory.txt` produced by `index_repository` or update scripts are not unauthorized.
- Regeneration of `.codebase-memory/SEMANTIC_MEMORY.md` and related docs by an approved update script is not unauthorized.
- The presence of `.codebase-memory` files in the working tree is not an integrity violation if they are ignored or generated by the MCP.

## 5. What Shall Always Be Regenerated by MCP

- `.codebase-memory/graph.db.zst` must be generated only by `codebase-memory` MCP `index_repository` with `persistence: true`.
- `.codebase-memory/artifact.json` is produced as a side effect of that indexing.
- Semantic markdown outputs may be regenerated by the MCP or by approved update scripts; they must not be fabricated manually.

## 6. Operational Workflow

1. Before work, consult `.codebase-memory/SEMANTIC_MEMORY.md` and `.codebase-memory/VALIDATION_REPORT.md`.
2. If knowledge is missing, prefer `search_graph` / `query_graph` over repository rescanning.
3. If a rescan is authorized, use `index_repository` with `persistence: true`.
4. Do not stage or commit `.codebase-memory` files in routine work.
