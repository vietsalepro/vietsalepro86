# VietSalePro — Codebase Memory Git Policy

> **Date**: 2026-07-24
> **Scope**: `.codebase-memory/` and `memory-zone/.codebase-memory/`

## 1. Policy Statement

`.codebase-memory` is a generated AI Workspace. It shall not be tracked by Git by default. This policy defines what must be ignored, what may be committed as an exception, and the migration path from the current mixed state.

## 3. File-by-File Git Treatment

| File | Treatment | Justification |
|---|---|---|
| `.codebase-memory/.gitattributes` | Untrack / ignore | Generated by MCP. Only purpose is to suppress merge conflicts on the binary artifact. If `.codebase-memory` is ignored, this file is not needed. |
| `.codebase-memory/artifact.json` | Untrack / ignore | Machine-generated index metadata (`schema_version`, `indexed_at`, `nodes`, `edges`). |
| `.codebase-memory/graph.db.zst` | Untrack / ignore | 7.8 MB compressed binary (43 MB original). Regenerated by `index_repository`. Causes merge conflicts and repo bloat. |
| `.codebase-memory/SEMANTIC_MEMORY.md` | Untrack / optionally relocate | AI-generated semantic summary. If a committed baseline is required, move to `docs/codebase-memory/SEMANTIC_MEMORY.md` or `ADMIN_DASHBOARD_PLAN_FIX_SPB/`. |
| `.codebase-memory/VALIDATION_REPORT.md` | Untrack / optionally relocate | AI-generated correction layer. Relocate if retained as governance record. |
| `.codebase-memory/CODEBASE_MEMORY_BASELINE.md` | Untrack / optionally relocate | AI-generated baseline registration. Relocate if retained. |
| `.codebase-memory/update-codebase-memory.txt` | Untrack / ignore | AI-generated update prompt artifact. |
| `memory-zone/.codebase-memory/*` | Untrack / ignore | Same generated workspace pattern under a different path. `memory-zone/` is already ignored. |

## 4. `.gitignore` Recommendation

Add the following lines to `.gitignore` (do not implement during this governance program):

```gitignore
# Codebase Memory AI workspace (generated by codebase-memory MCP)
.codebase-memory/
```

`memory-zone/` is already ignored by the existing `/memory-zone/` entry.

## 5. Migration Plan

1. Decide on curated baseline docs. If `SEMANTIC_MEMORY.md`, `VALIDATION_REPORT.md`, and `CODEBASE_MEMORY_BASELINE.md` are needed as committed records, move them to `ADMIN_DASHBOARD_PLAN_FIX_SPB/` or `docs/` and rename with a date/version suffix.
2. Add ignore rule. Insert `.codebase-memory/` into `.gitignore`.
3. Untrack generated files:
   `git rm --cached .codebase-memory/.gitattributes .codebase-memory/artifact.json .codebase-memory/graph.db.zst`
4. Verify. `git status` should show `.codebase-memory/` ignored and no tracked generated artifacts.
5. Commit the policy change. A single commit containing only `.gitignore` and the relocated docs, with Production Owner authorization.

## 6. Expected Impact

- Removes ~7.8 MB binary churn from the repository.
- Prevents accidental commits of regenerated artifacts.
- Eliminates merge-conflict risk on `graph.db.zst`.
- Preserves curated governance docs if relocated.

## 7. Risks

- Agents must remember that `.codebase-memory` is not committed; the MCP regenerates it per session or as requested.
- A clean `git status` may initially show ignored `.codebase-memory` files unless the index is also cleaned.
