# VietSalePro — Codebase Memory Governance Decision Record

> **Program**: Codebase Memory Governance Baseline Program
> **Date**: 2026-07-24
> **Status**: DECISIONS RECORDED

## Decision A — Classification of `.codebase-memory`

**Decision**: `.codebase-memory` is classified as an **AI Workspace** (containing Generated Runtime Artifacts and Documentation Artifacts).

**Justification**:

- The `codebase-memory` MCP binds the workspace to the repository root (`list_projects` returns `root_path: C:/PROJECT/vietsalepro`).
- Files are produced by `index_repository` (`graph.db.zst`, `artifact.json`) and update scripts (`SEMANTIC_MEMORY.md`, `VALIDATION_REPORT.md`).
- `.codebase-memory/.gitattributes` is auto-generated and marks `graph.db.zst` as binary.
- It is not human-authored source code, not repository metadata, and not a documentation workspace by default because the files are regenerated.

**Evidence**: `SEMANTIC_MEMORY.md`, `VALIDATION_REPORT.md`, `CODEBASE_MEMORY_BASELINE.md`, `artifact.json`, `.gitattributes`, `list_projects` output.

## Decision B — Version Control of `.codebase-memory` Files

**Decision**: By default, **no `.codebase-memory` file shall be under Git version control**. Markdown baselines may be committed only if relocated to a curated governance or docs directory and explicitly authorized.

**Justification**:

- `graph.db.zst` (7.8 MB compressed / 43 MB original) is a binary artifact that changes with every indexing.
- `artifact.json` is machine-generated metadata.
- Markdown docs are generated outputs, not authoritative source; committing them in `.codebase-memory` creates source-of-truth drift.
- If snapshot retention is required, the files must be moved out of the generated workspace and versioned under a human-controlled path.

**Evidence**: `git ls-files` shows `.codebase-memory/.gitattributes`, `artifact.json`, `graph.db.zst` tracked; `git status` shows `.codebase-memory/*.md` untracked and two binary files modified; `.gitignore` has no `.codebase-memory` entry.

## Decision C — Repository Inspection Policy

**Decision**: Repository inspection shall **ignore** `.codebase-memory` runtime-generated artifacts. While they remain tracked or modified, they are treated as governance exceptions, not source-code errors.

**Justification**:

- Treating generated artifacts as errors blocks legitimate MCP regeneration and creates false inspection failures.
- The WAVE-01 Repository Finalization Report already identified `.codebase-memory` modifications as a block.

**Evidence**: `WAVE-01_REPOSITORY_FINALIZATION_REPORT.md` §H; current `git status` showing modified `.codebase-memory` files.

## Decision D — Repository Finalization Policy

**Decision**: `.codebase-memory` generated artifacts shall **block** finalization if they are tracked or modified, unless they are ignored, reverted, or explicitly authorized by the Production Owner.

**Justification**:

- Finalization commits must represent approved source and governance state, not regenerated AI caches.
- The current finalization was `BLOCKED` because `.codebase-memory` files were modified/untracked.

**Evidence**: `WAVE-01_REPOSITORY_FINALIZATION_REPORT.md` §E, §H, §J.

## Decision E — Engineering Execution Agent Policy

**Decision**:

- Agents may read `.codebase-memory` docs; may update semantic markdown only under explicit incremental-update instruction; may never hand-edit binary artifacts.
- Agents may not commit `.codebase-memory` files unless explicitly authorized.
- MCP-generated changes are never unauthorized.
- Binary graph and artifact must always be regenerated by the `codebase-memory` MCP.

**Justification**:

- Prevents source-of-truth drift and merge conflicts while preserving legitimate MCP regeneration.
- The MCP is the only sanctioned producer of `graph.db.zst` and `artifact.json`.

**Evidence**: `CODEBASE_MEMORY_BASELINE.md` §7 update policy; `update-codebase-memory.txt`; MCP `index_repository` tool description.

## Decision F — ChatGPT Prompt Policy

**Decision**: Future prompts shall describe `.codebase-memory` as a generated, root-bound AI Workspace that is not committed by default and not relocatable.

**Justification**:

- Standard wording prevents repeated relocation attempts and accidental commit instructions.
- Provides a single reusable governance reference.

**Evidence**: `CODEBASE_MEMORY_RELOCATION_REPORT.md` outcome `BLOCKED`; `CODEBASE_MEMORY_RELOCATION_AUDIT.md` §3.2; `list_projects` output.
